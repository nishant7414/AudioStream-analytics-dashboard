# AudioStream Analytics Dashboard

A React dashboard that analyzes user listening behavior for an audio streaming platform (e.g. PocketFM-style). Built with **React**, **Tailwind CSS**, and **Recharts**.

## Features

- **KPI cards**: Total Users, Daily Active Users, Monthly Active Users, Average Listening Time, Retention Rate
- **Charts**:
  - Line chart – DAU trend over time
  - Bar chart – Top 10 shows by sessions
  - Pie chart – New vs Returning users
  - Bar chart – Listening time by device type
  - Funnel chart – App Open → Play → Complete Episode → Return
- **Filters**: Date range, device type, user type
- **Dark/Light mode** toggle (preference saved in localStorage)
- **Auto Insights**: Most popular show, average listening time, retention rate, total users
- **Responsive** layout and analytics-style UI

## Tech Stack

- React 18
- Vite
- Tailwind CSS
- Recharts

## Dataset

Synthetic dataset of **10,000 rows** with:

- `user_id`, `session_id`, `date`, `show_name`, `episode_id`
- `listening_time_minutes`, `completed_episode`, `device_type`, `user_type`

Data is generated by a Node script and saved as `public/data.json`.

## Step-by-step: Run locally

### 1. Prerequisites

- **Node.js** (v18 or newer recommended)  
  Check: `node -v`

- **npm** (comes with Node)  
  Check: `npm -v`

### 2. Open project folder

```bash
cd path/to/audiostream-analytics-dashboard
```

(Replace with your actual project path, or open the folder in your editor/terminal.)

### 3. Install dependencies

```bash
npm install
```

### 4. Generate the dataset (required before first run)

```bash
npm run generate-data
```

This creates `public/data.json` with 10,000 synthetic rows.  
If you skip this step, the app will show a message asking you to run this command.

### 5. Start the development server

```bash
npm run dev
```

Vite will start and show a local URL (e.g. `http://localhost:5173`). Open that URL in your browser.

### 6. Build for production (optional)

```bash
npm run build
```

Output is in the `dist` folder. Preview the build:

```bash
npm run preview
```

## Regenerating data

To create a fresh dataset (e.g. after changing the script):

```bash
npm run generate-data
```

Then refresh the app in the browser.
